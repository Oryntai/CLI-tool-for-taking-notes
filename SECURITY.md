# Security Policy

## Supported Versions

This is a small actively maintained project. Security fixes are applied to the latest `main` branch first.

## Reporting a Vulnerability

If you discover a security issue, please do not open a public issue first.

Instead, report it privately via GitHub Security Advisories:

1. Open the repository on GitHub.
2. Go to `Security` -> `Advisories`.
3. Create a new draft advisory with reproduction steps and impact.

If that flow is unavailable, open a minimal issue without exploit details and ask for a private contact channel.

## What to Include

- Affected version/commit
- Attack scenario and impact
- Reproduction steps
- Suggested fix (if available)

## Response Targets

- Initial triage: within 7 days
- Status update: within 14 days
- Fix timeline: depends on severity and complexity
